# AI-Researcher PRD

> 构建一个“从研究问题到论文草稿”的科研智能助手，支持按主题与约束条件自动检索、阅读、分析文献，并生成可验证、可复用的研究方案与论文初稿。

------

# 一、用户输入（Research Intent）

### 核心输入项

- **主题关键词**（必填）
- **发表时间范围**
  - 起始年 – 结束年
- **期刊 / 会议水平**
  - 顶级 / 一区 / 二区 / 不限
- **文献类型**
  - 综述（Survey / Review）
  - 原创研究（Research / Method）
- **学科方向（可选）**
  - CV / NLP / Systems / ML / Bio / 交叉

👉 输出：一个**结构化 Research Intent 对象**

------

# 二、文献检索与筛选（Literature Discovery）

### 系统能力

- 多源文献搜索（抽象能力，不限定数据源）
- 按以下维度过滤：
  - 年份
  - 期刊/会议等级
  - 是否综述
  - 关键词相关度
- 自动去重、合并版本（arXiv / Journal）

### 输出

- 文献候选池（Top N）
- 每篇文献的：
  - 标题
  - 年份
  - 期刊/会议
  - 类型（综述 / 方法）
  - relevance score

------

# 三、文献阅读与理解（Paper Reading Engine）

### 系统做什么

对每篇论文自动完成：

- 摘要级理解
- 方法级理解
- 实验设置理解
- 贡献点提取
- 局限性/问题提取

### 输出（结构化）

对每篇论文生成：

- 核心问题
- 关键方法
- 技术路线
- 实验结论
- 明确不足

👉 **不是生成长文本，而是可被后续模块消费的结构信息**

------

# 四、研究脉络与空白分析（Research Landscape）

### 系统能力

- 自动聚类文献方向
- 梳理技术演进路线
- 对比不同方法的假设、模型、实验
- 明确：
  - 已解决的问题
  - 半解决的问题
  - 明显未覆盖的问题

### 输出

- 研究方向地图（文字结构）
- Top 未解决研究问题列表（ranked）

------

# 五、研究想法生成（Idea Generation）

### 系统能力

基于：

- 文献不足
- 方法冲突
- 实验缺口

生成多条候选研究想法：

- 每条想法包含：
  - 研究动机
  - 核心假设
  - 预期贡献
  - 与现有方法的区别

### 用户可选

- 选择 / 编辑 / 合并想法

------

# 六、算法 / 模型设计（Method Design）

### 系统能力

将“研究想法”转化为：

- 整体模型框架描述
- 核心模块拆分
- 与 baseline 的差异点
- 理论 / 直觉解释

> **不是写代码，是写“论文中 Method 部分的逻辑结构”**

### 输出

- 方法概述
- 模型流程描述
- 模块功能说明

------

# 七、实验设计与结果分析（Experiment Planning）

### 系统能力

- 自动生成：
  - 实验设置建议
  - baseline 对比方案
  - ablation 设计
- 在没有真实实验结果时：
  - 生成**合理、可解释的“占位型结果分析”**
  - 明确哪些是“假设结果”

### 输出

- 实验表结构
- 结果分析草稿
- 风险点说明

------

# 八、论文草稿生成（Paper Drafting）

### 论文结构支持

- Abstract
- Introduction
- Related Work
- Method
- Experiments
- Discussion / Limitation
- Conclusion

### 特点

- 明确区分：
  - 来自文献的事实
  - 推导出的研究假设
  - 用户原创想法
- 语言符合学术风格，但**不伪造实验**

------

# 九、系统级原则（你可以原样给团队）

1. **模块解耦**
   - 检索 ≠ 阅读 ≠ 想法 ≠ 写作
2. **所有中间结果结构化**
   - 不是“让模型自由发挥”
3. **人始终在关键决策点**
   - 选文献
   - 选研究想法
4. **允许不完整**
   - 实验可以是假设
   - 明确标注
5. **不追求“一键出论文”**
   - 追求“科研加速器”

# 

> 我们要做的是一个科研工作流系统，而不是一个大模型 demo。
> 所有功能围绕“从研究问题 → 研究方案 → 论文结构”的可控推进。
